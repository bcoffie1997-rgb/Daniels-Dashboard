# 05 — QA

The goal isn't exhaustive testing. It's catching the things that, if they break in front of a chef at Daniel's, get the app banned from the kitchen. Prioritize the critical path.

## Critical path manual test checklist

Must pass on **both iPhone Safari and Android Chrome** before any pilot.

### Auth
- [ ] `/` signed out → redirects to `/login`
- [ ] Valid email → success state, no error details leaked
- [ ] Invalid email format → inline validation error
- [ ] Magic link click → lands on `/`, session active
- [ ] Expired magic link → graceful error, prompt for new
- [ ] Sign out → cleared, back to `/login`
- [ ] Counter visits `/admin/*` → `/403`
- [ ] Manager visits `/admin/users` → `/403`
- [ ] Admin → full access

### Admin: catalog
- [ ] Create station with empty name → validation error
- [ ] Create station with valid data → appears immediately
- [ ] Drag-reorder stations → persists after refresh
- [ ] Archive station → disappears from counter picker; historical sessions preserved
- [ ] Create item with each unit type (lb, oz, each, bottle, case, gal, qt, l, ml, kg, g)
- [ ] Bulk import 20 valid rows → preview shows 20, import succeeds
- [ ] Bulk import mix of valid + invalid → invalid highlighted, only valid imported
- [ ] Promote user to manager → gains dashboard access on next page load
- [ ] Demote admin to counter → loses /admin access on next request

### Counter flow (online)
- [ ] Station picker shows only active stations
- [ ] "Last counted" timestamp accurate
- [ ] Tap station → exactly one in_progress session created
- [ ] Tap same station → resumes session, no duplicate
- [ ] Item list in correct sort_order
- [ ] "Previous: X {unit}" displays for items with history
- [ ] No "Previous" shown for never-counted items
- [ ] Tap item → drawer opens with focus
- [ ] OS keypad shows decimal point (iOS Safari + Android Chrome)
- [ ] Enter 0 → saves as 0 (not empty)
- [ ] Enter decimal (3.5) → saves correctly
- [ ] "Same as last" pre-fills with previous
- [ ] Skip → item flagged warm gold, advances
- [ ] Progress counter increments
- [ ] "Save & Continue Later" → session remains in_progress
- [ ] Resume in_progress → all prior entries present
- [ ] Review screen flags items with >20% variance
- [ ] Submit → status flips to submitted, redirected to detail

### Counter flow (offline) — THE CRITICAL TEST
- [ ] Airplane mode → indicator shows offline state (sage dot)
- [ ] Start session offline → works
- [ ] Enter 10 entries offline → all visible
- [ ] Submit offline → marked submitted locally, indicator shows pending
- [ ] Force-close PWA → reopen → data still present
- [ ] Re-enable network → within 30s, sync completes
- [ ] Verify in Supabase: one session, 10 entries, status=submitted
- [ ] Trigger sync twice (manual retry) → no duplicate rows (idempotency)
- [ ] **Test in an actual Daniel's walk-in before merging Sprint 4**

### Manager
- [ ] Dashboard shows submitted sessions newest-first
- [ ] Filters (date, station, counter, status) work and persist in URL
- [ ] Session detail variance computed correctly
- [ ] Variance table sorted by |variance_pct| desc by default
- [ ] Approve → status flips, approver stamped
- [ ] Reject without reason → button disabled
- [ ] Reject with reason → status flips, reason persisted
- [ ] Counter sees rejection notification in real time
- [ ] Variance report identifies chronically variable items

### CSV export
- [ ] Export with date range → downloads file with correct name
- [ ] Opens cleanly in Excel + Google Sheets — columns aligned
- [ ] Notes containing commas → properly quoted
- [ ] Notes containing newlines → handled correctly
- [ ] Empty result set → header-only CSV, no error

## Edge cases

| Case | Expected behavior |
|---|---|
| Item archived mid-count | Still appears in this session, marked "archived" tag, can still enter quantity |
| Item renamed mid-count | New name appears on next refresh |
| Two devices, same counter, same station | One session wins (first created), second device picks it up |
| Submit session with zero entries | Blocked at UI; if forced via API, server rejects |
| 0% variance (counted exactly same) | No flag, displays cleanly |
| Negative variance (-100%, counted zero) | Flagged terracotta, prompt for note |
| Previous was zero, current non-zero | Show as "+X" not "∞%" — handle divide-by-zero |
| Manager approves already-approved session | Idempotent, no double-stamp |
| Counter signs out mid-count | Session remains in_progress, resumable |
| Demoted from manager → counter while on dashboard | Next action returns 403 |
| PWA updated, stale service worker | Skip-waiting + reload prompt: "New version available — tap to update" |
| Large export (6+ months, thousands of entries) | Streams without timeout |
| iOS PWA closed 2 weeks, reopened | Session refresh works |
| DST transition during a count | started_at / submitted_at as UTC; display America/New_York correctly |

## Security checklist

- [ ] RLS enabled on every public table — verify with: `SELECT relname FROM pg_class WHERE relrowsecurity = false AND relkind = 'r' AND relnamespace = 'public'::regnamespace;` returns nothing
- [ ] Every RLS policy tested at all three roles
- [ ] Service role key never exposed to client (audit `NEXT_PUBLIC_*` vars)
- [ ] Server actions validate role server-side, never trust client
- [ ] Zod validation on every form input
- [ ] Only parameterized queries / Supabase client — no raw SQL with user input
- [ ] Magic link email rate-limited (Supabase default; verify not disabled)
- [ ] CSP header set in `next.config.mjs`
- [ ] No `console.log` of sensitive data in production builds
- [ ] Sentry redacts PII in payloads
- [ ] PostHog `person_profiles` set appropriately — no PII in event properties
- [ ] Audit log captures all role changes
- [ ] `pnpm audit` shows no high/critical vulns
- [ ] Supabase account: 2FA enabled
- [ ] GitHub repo: branch protection on `main`, required PR review
